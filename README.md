歡迎來到 Python 基礎程式設計課程！本課程專為初學者設計，從零開始建立您的 Python 程式設計能力。

## 📚 課程結構

### 學習單元（按順序學習）

```
Unit 1 → Unit 2 → Unit 3 → Unit 4 → Unit 5 → Unit 6
   ↓        ↓        ↓        ↓        ↓        ↓
 變數    運算子   輸入輸出  條件判斷   迴圈    函式
```

#### **Unit 1: 變數與資料型態**

- 學習目標：理解變數概念，掌握四種基本資料型態（int, float, str, bool）
- 核心技能：變數宣告、型態檢查、型態轉換

#### **Unit 2: 運算子與表達式**

- 學習目標：掌握算術、比較、邏輯三大運算子類別
- 核心技能：運算子優先順序、表達式求值、短路求值

#### **Unit 3: 輸入輸出**

- 學習目標：學習與使用者互動（input, print, f-string）
- 核心技能：格式化輸出、使用者輸入處理

#### **Unit 4: 條件判斷**

- 學習目標：掌握 if/elif/else 語法，實作程式分支邏輯
- 核心技能：條件表達式、巢狀條件、邏輯組合

#### **Unit 5: 迴圈控制**

- 學習目標：掌握 for 和 while 迴圈，避免重複程式碼
- 核心技能：range() 函式、break/continue、巢狀迴圈

#### **Unit 6: 函式設計**

- 學習目標：學會定義與呼叫函式，模組化程式碼
- 核心技能：函式定義、參數傳遞、回傳值、作用域

## 🚀 如何使用本課程

### 步驟 1：準備環境

**必要軟體**：

- Python 3.8 或更高版本
- Jupyter Notebook（建議使用 Anaconda 或 VS Code + Jupyter 擴充功能）

**檢查安裝**：

```python
# 在終端機執行
python --version  # 應顯示 Python 3.8+

# 或在 Python 互動環境中
import sys
print(sys.version)
```

### 步驟 2：學習流程

#### 對於每個單元（Unit 1-6）：

1. **閱讀講義**

   - 開啟 `01-lecture.ipynb`
   - 仔細閱讀理論基礎部分
   - 理解核心概念與範例
2. **動手實作**

   - 在講義中執行每個程式碼範例
   - 嘗試修改範例，觀察結果變化
   - 完成練習題
3. **複習與檢核**

   - 確認自己理解每個學習目標
   - 嘗試解釋核心概念（可以寫下筆記）
   - 如有疑問，標記起來後續詢問

#### 完成專案（Milestone）：

1. **閱讀需求**

   - 先讀 `README.md` 了解專案目標
   - 詳細閱讀 `requirements.ipynb` 的功能規格
2. **規劃設計**

   - 理解輸入、處理、輸出三個環節
   - 規劃需要的變數與程式結構
3. **開始實作**

   - 使用 `starter-code.ipynb` 作為起始點
   - 依照 TODO 提示逐步完成
   - 每個功能完成後先測試
4. **測試與優化**

   - 使用提供的測試案例驗證功能
   - 檢查程式碼品質（命名、註解、格式）
   - 參考 `solution.ipynb`（先嘗試自己完成！）

## 📝 學習建議

### 1. 循序漸進

- **嚴格按照順序學習**：每個單元都建立在前一個單元的基礎上
- 不要跳過單元，確保理解每個概念再進行下一步

### 2. 動手實作

- **程式設計需要練習**：只看不寫永遠學不會
- 每個範例都要自己執行一次
- 嘗試修改程式碼，觀察結果變化

### 3. 理解原理

- 不要死記語法，理解「為什麼」這樣寫
- 嘗試解釋每個概念（可以寫筆記或講給別人聽）

### 4. 除錯技巧

- **錯誤是學習機會**：遇到錯誤時，先仔細閱讀錯誤訊息
- 使用 `print()` 檢查變數值
- 將複雜問題拆解成小問題

### 5. 尋求幫助

- 遇到困難時，先嘗試自己解決（查文件、重新閱讀講義）
- 如果卡住超過 20 分鐘，尋求協助（同學、助教、老師）
- 描述問題時，提供：錯誤訊息、相關程式碼、您已經嘗試過的方法

## ❓ 常見問題（FAQ）

### Q1: 我完全沒有程式設計經驗，可以學嗎？

**A**: 可以！本課程專為初學者設計，從最基礎的概念開始。只要您有：

- 基本的電腦操作能力
- 邏輯思考能力
- 願意花時間練習

就能順利完成課程。

### Q2: 需要什麼樣的電腦？

**A**:

- **作業系統**：Windows、macOS、Linux 都可以
- **記憶體**：建議 4GB 以上
- **儲存空間**：至少 1GB 可用空間（安裝 Python 和套件）

### Q3: 每個單元要花多少時間？

**A**:  實際時間會因個人學習速度而有所不同, **重點是理解，不是速度**。

### Q4: 可以使用其他編輯器嗎？

**A**: 可以！但建議使用 Jupyter Notebook，因為：

- 可以分步驟執行程式碼
- 方便記錄筆記和心得
- 課程範例都是 Jupyter Notebook 格式

如果您熟悉其他編輯器（如 VS Code、PyCharm），也可以使用，但需要自行轉換格式。

### Q5: 遇到錯誤怎麼辦？

**A**:

1. **仔細閱讀錯誤訊息**：Python 的錯誤訊息通常會告訴您問題在哪裡
2. **檢查拼字**：變數名稱、函式名稱是否正確
3. **檢查縮排**：Python 對縮排很敏感
4. **檢查型態**：確認變數的型態是否符合預期
5. **使用 print() 除錯**：在關鍵位置輸出變數值

如果還是無法解決，記下錯誤訊息和相關程式碼，尋求協助。

### Q6: 可以看解答嗎？

**A**:

- **建議流程**：先自己嘗試完成，真的卡住再看解答
- **看解答時**：不要只是複製貼上，要理解為什麼這樣寫
- **看過解答後**：蓋起來，自己重新寫一次

### Q7: 如何知道自己已經學會了？

**A**: 您可以：

- 嘗試解釋核心概念給別人聽
- 不看講義，自己完成練習題
- 能夠修改範例程式碼來解決新問題
- 能夠獨立除錯和解決問題

## 💡 學習心法

> **「程式設計不是看會的，而是寫會的。」**

1. **不要害怕錯誤**：錯誤是學習的一部分，每個程式設計師都會犯錯
2. **多練習**：每寫一行程式碼，就加深一點理解
3. **保持好奇心**：嘗試修改程式碼，看看會發生什麼
4. **建立信心**：從簡單的程式開始，逐步建立成就感
5. **享受過程**：程式設計是一門創造性的活動，享受解決問題的樂趣！
