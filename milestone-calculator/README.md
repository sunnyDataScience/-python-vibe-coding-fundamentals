## 📋 專案描述

### 情境說明

在這個專案中，您將開發一個簡單的命令列計算機程式。使用者可以輸入兩個數字和一個運算符號，程式會計算並顯示結果。

**範例**：

```
請輸入第一個數字: 10
請輸入運算符號 (+, -, *, /): +
請輸入第二個數字: 5
10.0 + 5.0 = 15.0
```

### 功能需求

#### 基本需求（必須完成）

1. **輸入功能**

   - 提示使用者輸入第一個數字
   - 提示使用者輸入運算符號（+, -, *, /）
   - 提示使用者輸入第二個數字
2. **計算功能**

   - 支援四則運算：加（+）、減（-）、乘（*）、除（/）
   - 正確處理浮點數運算
3. **輸出功能**

   - 以易讀格式顯示計算結果
   - 例如：`10 + 5 = 15`

#### 進階需求（選做）

4. **錯誤處理**

   - 檢查除數是否為零
   - 檢查運算符號是否有效
5. **擴展功能**

   - 支援更多運算：次方（**）、餘數（%）、整數除法（//）
   - 支援連續計算（重複使用結果）

### 使用的技術

您將使用以下技術來完成這個專案：

- **Python 版本**：3.8 或更高版本
- **主要技術**：
  - 變數與資料型態（int, float, str）
  - 算術運算子（+, -, *, /）
  - 輸入輸出（input(), print(), f-string）
  - 條件判斷（if/elif/else，如果完成完整版）
- **預計程式碼行數**：約 20-40 行（依據功能而定）

不知道從哪裡開始？別擔心！跟著下面的步驟，一步一步完成您的計算機程式。

### Phase 1: 分析與設計（約 15 分鐘）

先不要急著寫程式碼，先理解要做什麼很重要！

#### 步驟 1：理解需求

#### 步驟 2：設計資料流

```
使用者輸入 → 型態轉換 → 執行運算 → 顯示結果
```

### Phase 2: 核心功能實作（約 30-45 分鐘）

現在開始動手寫程式碼！不要害怕犯錯，每個程式設計師都是從錯誤中學習的。

#### 步驟 4：實作輸入模組

**參考**：`starter-code.ipynb` 提供註解形式的程式碼框架和 TODO 提示，`requirements.ipynb` Part I 需求 1.1 有詳細說明

**範例程式碼**：

```python
# 提示使用者輸入並轉換型態
num1 = float(input("請輸入第一個數字: "))
operator = input("請輸入運算符號 (+, -, *, /): ")
num2 = float(input("請輸入第二個數字: "))
```

**提示**：`starter-code.ipynb` 中的 TODO 1 提供註解形式的範例，您可以取消註解或自行撰寫程式碼。

**小提醒**：

- 為什麼使用 `float()` 而非 `int()`？

  - 因為 `float()` 可以處理整數和小數（如 3.5），而 `int()` 只能處理整數
  - 這樣使用者可以輸入小數進行運算
- 如果使用者輸入非數字會發生什麼？

  - 會產生 `ValueError` 錯誤
  - 基礎版可以先不管這個，進階版可以加入錯誤處理（參考 `solution.ipynb`）

#### 步驟 5：實作計算邏輯

**參考**：`starter-code.ipynb` 提供 TODO 提示和程式碼框架（註解形式），完整解答請參考 `solution.ipynb`

**基礎版（只支援加法）**：

```python
# 如果還不會條件判斷，可以先只支援加法
result = num1 + num2
```

**完整版（支援四則運算，需要條件判斷，Unit 4 才會教）**：

```python
if operator == "+":
    result = num1 + num2
elif operator == "-":
    result = num1 - num2
elif operator == "*":
    result = num1 * num2
elif operator == "/":
    result = num1 / num2
else:
    print("錯誤：不支援的運算符號！")
```

**提示**：`starter-code.ipynb` 中的 TODO 2 提供註解形式的程式碼框架作為參考，您可以取消註解並理解邏輯。如果還不會條件判斷，可以先只做加法運算。

#### 步驟 6：實作輸出模組

**參考**：`requirements.ipynb` Part I 需求 1.3 有格式化範例，`starter-code.ipynb` 的 TODO 3 提供註解形式的範例

```python

```

**提示**：`starter-code.ipynb` 中的 TODO 3 提供註解形式的輸出範例，您可以取消註解或自行撰寫程式碼。

## ❓ 常見問題（FAQ）

### Q1: 我不知道如何根據運算符號選擇不同的運算

**建議**：

- **基礎版**：如果還不會條件判斷（if/elif/else），可以先只支援加法運算
- **完整版**：`starter-code.ipynb` 的 TODO 2 提供註解形式的 if/elif 模板作為參考
- **學習資源**：條件判斷會在 Unit 4 詳細學習，現在可以先參考 `solution.ipynb` 中的範例

**提示**：您可以先取消 `starter-code.ipynb` 中註解形式的程式碼，觀察它的運作方式，理解後再自己寫一次。

### Q2: 我執行程式時，`10 + 5` 的結果是 `105` 而不是 `15`

**問題原因**：忘記將 `input()` 的結果轉換為數字型態

- `input()` 回傳的是字串（str），所以 `"10" + "5"` 會是字串連接，得到 `"105"`
- 需要使用 `float()` 將字串轉換為數字：`float(input("請輸入數字: "))`

**參考資源**：`requirements.ipynb` Part I 需求 1.1 有詳細說明與錯誤範例

### Q3: 除以零的錯誤處理太難了

**建議**：

- **基礎版**：可以先不做錯誤處理，只要能完成基本功能即可
- **進階版**：參考 `solution.ipynb` 進階版，裡面有完整的錯誤處理範例

**提示**：您可以在執行除法前檢查 `num2` 是否為 0：

```python
if operator == "/":
    if num2 == 0:
        print("錯誤：不能除以零！")
        # 結束程式或返回
    else:
        result = num1 / num2
```

### Q4: 我不知道從哪裡開始

**建議步驟**：

1. **先閱讀需求**：打開 `requirements.ipynb`，了解這個專案要做什麼
2. **使用起始框架**：打開 `starter-code.ipynb`，按照 TODO 提示逐步完成
3. **先做基礎版**：如果覺得困難，可以先完成只支援加法的版本
4. **測試程式**：每完成一個功能就測試一次，確認運作正常
5. **參考解答**：如果卡住了，可以參考 `solution.ipynb`，但建議先自己嘗試
